02

Raw Text Extraction — extractors/*.py

Each extractor converts its file type into a standard list of page-text dicts. Same output format regardless of input.

PARALLEL LANES OCR INTENSIVE
▼
LANE A — TEXT PDF (pdfplumber)
1

Open PDF with pdfplumber

Reads all pages. Handles multi-column layouts, rotated pages, embedded fonts.

2

Extract prose text per page

Preserves paragraph structure. Strips headers/footers if detected.

3

Extract tables → pipe-delimited text

Financial tables converted to col1 | col2 | col3 so the LLM can parse them naturally.

4

Output: List of page dicts

{"page":1, "text":"...", "has_tables":true}
LANE B — SCANNED PDF (OCR)
1

Convert pages → high-res images (300 DPI)

pdf2image + poppler. Higher DPI = better OCR but slower.

2

Pre-process images

Grayscale → Sharpen → Contrast boost. Removes scan artifacts that confuse OCR.

3

Run Tesseract OCR per page

PSM 6 (uniform text block). Lang: eng or eng+hin for bilingual docs.

4

Confidence scoring per page

If OCR confidence <60%, flag page as LOW confidence. Alert credit manager to re-scan at higher quality.

All extractors output the same format: [{"page": N, "text": "...", "source": "filename", "type": "pdf_text|pdf_scanned|docx|xlsx", "confidence": 95}] — this standardisation means the LLM layer never needs to know what file type it came from.

03

LLM Structuring — core/llm.py

The intelligence layer. Raw messy text → clean typed JSON with field-level confidence scores.

CLAUDE / GPT-4 JSON OUTPUT
▼
WHAT HAPPENS INSIDE THE LLM CALL
1

Combine all pages into one document string

All extracted pages merged. If total >80k chars, front 60% + back 40% is kept (financials are usually at the back).

2

Build the extraction prompt

System prompt defines the credit analyst role. User prompt contains the document text + target JSON schema with 12 required top-level keys.

3

LLM extracts and reasons over the text

Claude/GPT-4 reads the document, finds revenue figures, debt ratios, audit qualifications, red flags, and returns them as a typed JSON.

4

Every field gets source + confidence

{"value": 21893, "unit": "crore", "period": "FY2024", "confidence": "HIGH"}
5

Unknown fields go to fields_not_found[]

If bank statement not uploaded, GST data missing etc — listed explicitly so gaps are visible.

WHAT THE LLM EXTRACTS
✓

Financial figures

Revenue, EBITDA, PAT, debt, gearing — across all available periods

✓

Audit qualifications

Emphasis of matter, qualifications, auditor resignation, going concern doubts

✓

Contingent liabilities

Tax demands, legal claims, guarantees — even if buried in footnotes

✓

Red flags in narrative

"Circular trading", "diverted funds", "project delayed", "insolvency proceedings" — keyword + semantic detection

✓

Promoter / Director details

Names, stake percentages, disqualification flags, related party transactions

PRODUCTION: Multiple documents from the same company are sent to the LLM in sequence and results are merged — conflicts between sources are flagged (e.g., revenue in annual report differs from ITR). Each LLM call is ~5-15 seconds. With 10 documents per company and 50 companies, this is the main latency bottleneck → use async batching.

04

Cross-Validation Engine — validators/cross.py

Compares figures across multiple sources. This is where circular trading, hidden debt, and ITC anomalies are caught.

FRAUD DETECTION 10 CHECKS
▼
CV_001
CRITICAL IF FAIL
Interest Coverage Ratio
Fails if ICR < 1.5x. Company cannot service debt from operations.
CV_002
HIGH IF FAIL
Gearing Ratio Check
Warn if >1.0x. Critical if >2.0x. Indicates over-leveraged balance sheet.
CV_003
MEDIUM
PAT Margin Check
Warn if PAT margin < 2%. Very thin margins leave no buffer for downturns.
CV_004
HIGH IF FAIL
Debt Trend Analysis
Flag if debt grew >30% YoY. Rapid debt growth without revenue growth = danger.
CV_005
MEDIUM
Cash Trend Analysis
Warn if cash fell >30% over review period. Liquidity stress indicator.
CV_006
HIGH IF FAIL
Revenue Consistency
Flag if revenue grew >100% — may indicate inflation. Flag if fell >30%.
CV_007
CRITICAL
Audit Qualifications
Any audit qualification = immediate flag. Auditor resignation = CRITICAL.
CV_008
CRITICAL
Litigation Detection
Court mentions, NI Act cases, winding-up petitions, IBC proceedings.
CV_009
🚨 FRAUD FLAG
GST vs Bank Mismatch
If GST turnover > bank credits by 1.3x+ → Circular trading suspected. The killer check.
CV_010
HIGH IF FAIL
ITC Ratio vs Sector
If ITC claimed is 50%+ above sector benchmark → Fabricated input tax credits.
CV_009 + CV_010 only fire if GST + Bank data are both uploaded. Upload GSTR-3B CSV + bank statement CSV together to unlock these fraud checks. This is why multi-document upload is critical — no single document reveals circular trading.
